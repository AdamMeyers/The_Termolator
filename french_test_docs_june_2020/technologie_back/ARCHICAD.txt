ARCHICAD

ARCHICAD est un logiciel d'architecture édité par la société hongroise Graphisoft, du groupe allemand Nemetschek, qui permet de créer un modèle en 3D d'un bâtiment puis de créer divers documents nécessaires à sa construction (plan, perspective, ...).

Le logiciel permet de gérer autour d'une maquette numérique 3D tous les documents entrant dans les compétences d'un architecte tout au long du cycle de production d'un projet architectural : plans, détails, perspectives, imagerie de synthèse, métrés.
Avec son concept de Virtual Building ARCHICAD innove dès sa création dans ce qu'il est convenu d'appeler les logiciels BIM (Building Information Modeling)

ARCHICAD est disponible pour les systèmes d'exploitation Windows et Mac OS X.

ARCHICAD nécessite des machines robustes, de préférence multiprocesseurs 64 bit, comportant beaucoup de RAM et dotées de cartes graphiques performantes

Sur Mac, depuis la version 13, seules les machines MacIntel sont supportées.

Depuis la version 17, seules les machines équipées de multiprocesseurs 64 bit peuvent faire tourner le logiciel.

Des versions d’essai commerciale, éducative, et entièrement fonctionnelle de 30 jours peuvent être installées avec le même fichier d’installation. Tant qu’aucune protection matérielle n’est présente ou que le logiciel n’est pas activé avec un numéro de série de version d’essai ou éducative, ARCHICAD peut être lancé en mode démo. Les fichiers d’installation peuvent être téléchargés ici: http://www.graphisoft.com/downloads sans inscription. Les numéros de série des versions d’essai et éducative peuvent être obtenus ici MyarchiCAD.com après inscription.

"Virtual Building" est le concept, mis en avant par ARCHICAD. Il s'apparente au BIM (Building Information Modeling) dont ARCHICAD est ainsi l'un des pionniers.

ARCHICAD organise l'information autour d'une Maquette numérique 3D. Les informations sont encodées principalement à partir de la fenêtre de visualisation en plan, mais rien n'empêche de construire la maquette 3D directement à partir de la fenêtre de visualisation 3D ou à partir des coupes et des façades. Plus généralement le logiciel autorise l'édition de n'importe quel élément (Un mur ou une fenêtre p.ex.) à partir de tous les documents(perspective, axonométrie, coupes, façades, nomenclatures...) ouverts dans le projet courant. Les modifications sont répercutées dans tous les documents de manière automatique.

Le "Virtual Building", dans un autre volet, s'intéresse également aux formats de communication vers d'autres logiciels spécialisés auxquels correspondent d'autres métiers de la construction: Stabilité, techniques spéciales... Cette communication est notamment assurée par les IFC (Industry Foundation Classes) très largement proposés à tous les niveaux du logiciel (Import, Export, objets spécialisés, informations liées aux éléments du modèle).

Depuis la version 5.1 de ARCHICAD en 1997, il est possible de partager un fichier "via" "Teamwork" ce qui permet à plusieurs architectes de travailler sur le même projet. Avec la version 13 d'ARCHICAD en 2009, arrive une version "TeamWork 2.0" complètement réécrite, avec une nouvelle approche "base de données", nommée "BIM Serveur". Depuis lors, seuls les changements et différences sont communiqués à la base de données. Il est en plus possible de se connecter à distance à la base de donnée en passant par Internet, permettant ainsi de collaborer à un projet depuis différents pays.

Le GDL pour Geometric Description Language, un langage propriétaire de Graphisoft, s'apparente au BASIC, et permet de développer les objets 2D/3D utilisés par le programme
Les outils de modélisation sont : mur, dalle, poteau, poutre, toit ou maillage pour les terrains...
Une bibliothèque d'objets paramétrables basés sur le GDL est fournie avec ARCHICAD. Ces objets sont des fenêtres, des portes, du mobilier, des lampes...
Une nouvelle famille d'outils apparaît, les objets collectifs, tel l'outil mur rideau qui organise une multitude d'objets en système. L'outil mur rideau gère de grands ensembles de mur rideau.

De par la très large diffusion de AutoCAD à travers le monde, le format natif DWG d'AutoCAD et le format standard industriel DXF sont devenus de facto les formats obligés pour exporter vers d'autres logiciels CAO. ARCHICAD est membre de l'Open Design Alliance et contribue à l'élaboration d'une bibliothèque DWG ouverte.

Une API développée par Graphisoft, "Virtuel Building Explorer", permet de générer des fichiers 3D exécutables dans lesquels la maquette 3D, issue de ARCHICAD, peut être explorée à la façon d'un jeu vidéo

IFC, Industry Foundation Classes, est un format de fichier neutre qui rend possible l'échange d'informations entre différentes systèmes CAO.

Le format IFC mémorise plus que l'information 2D du modèle. Il incorpore les informations 3D et des informations plus abstraites (par exemple si un élément est porteur ou non porteur)

Le format IFC est certifié ISO et peut être intégré dans le système de contrôle qualité d'un bureau. IFC est développé en partie par l'Alliance internationale pour l'interopérabilité (IAI). Ce format est soutenu depuis 1996 par Graphisoft, le fabricant d'ARCHICAD.

Le Format des mises en pages dans PlotMaker était le ".LAY", Dans la version 9 de PlotMaker, ".LBK".

ARCHICAD, à partir de la version 10 ouvre le format LBK
